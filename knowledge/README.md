# knowledge/

Konzeptuelles und methodisches Fundament des Tools. Vier Dokumente, in Lesereihenfolge:

1. [00-projektkonzept.md](00-projektkonzept.md) — Frage, Ziele, Architektur, Vorgehen, Erfolgskriterien
2. [wissensnetzwerk-definition.md](wissensnetzwerk-definition.md) — drei konkurrierende Antworten auf die Frage, was ein Wissensnetzwerk ist
3. [methodik-triangulation.md](methodik-triangulation.md) — Konvergenz und Divergenz der drei Sichten, Schwellwerte mit Begründung
4. [methodik-validierung.md](methodik-validierung.md) — Befund, Diagnose, Hypothese; Privacy und Reproduzierbarkeit

Diese Dokumente sind nicht nachträgliche Dokumentation, sondern das Fundament, auf dem das Tool steht. Sie sind langfristig stabil und ändern sich nur, wenn sich die Position des Projekts ändert.

## Verhältnis zu METHODIK.md und README.md im Root

- `README.md` im Root: kurze Projektbeschreibung, Ausführungsanweisung, Output-Struktur
- `METHODIK.md` im Root: kompakte Begründung der epistemologischen Position für den Erst-Leser
- `knowledge/`: ausgearbeitete Vertiefungen einzelner Aspekte

Wer das Repo zum ersten Mal sieht, liest README → METHODIK → `knowledge/00-projektkonzept.md`. Wer die Methode prüfen will, liest die drei Methodik-Dokumente.
